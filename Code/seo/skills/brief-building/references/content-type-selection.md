# Content Type Selection Reference

Decision framework for choosing the right content format based on search
intent and SERP signals.

---

## Decision Tree

Match search intent and SERP signals to content type:

| Intent | Query Signal | SERP Dominant Format | Content Type |
|--------|-------------|---------------------|-------------|
| Informational | "what is", definition queries | Knowledge panels, short answers | Definition / explainer |
| Informational | "how to", process queries | Step-by-step results, video | How-to tutorial |
| Informational | Broad topic, multiple sub-topics | Long-form guides dominate | Pillar guide |
| Commercial | "best", "top", list queries | Listicle-style results | Listicle |
| Commercial | "vs", "or", comparison queries | Comparison tables, side-by-side | Comparison |
| Mixed | Broad + recent interest | News articles, opinion pieces | News / opinion |
| Mixed | Broad topic, varied results | No dominant format | Match top performer or fill gap |

### When to Match the SERP

Match the dominant format when:
- Top 3-5 results all use the same format
- Those results are ranking well (high traffic, strong positions)
- The format aligns with the query intent

### When to Differentiate

Differentiate when:
- All results are shallow → go deeper (more comprehensive)
- All results are long → provide a concise, scannable alternative
- All results miss a key angle → fill the gap
- All results are dated → provide fresh, current coverage
- Quality is uniformly low → outclass with better research and structure

## Outline Templates by Content Type

### Pillar Guide

Target: 2,500-4,000 words (adjust based on competitor analysis)

```
H1: [Primary keyword]: [Comprehensive hook]

H2: [Quick answer / TL;DR]
    → 200-300 words. Direct answer for Featured Snippet and skimmers.
    → Front-load the answer to the title question.

H2: [Core sub-topic 1 — from top PAA question or competitor pattern]
    H3: [Detail or sub-question]
    H3: [Detail or sub-question]
    → 600-1,000 words per core section

H2: [Core sub-topic 2]
    H3: [Detail]
    H3: [Detail]

H2: [Core sub-topic 3]
    H3: [Detail]
    H3: [Detail]

H2: [Practical application / How to use this]
    → 300-500 words. Actionable steps.

H2: Frequently Asked Questions
    H3: [PAA question 1]
    H3: [PAA question 2]
    H3: [PAA question 3]
    → Schema-ready Q&A format. 3-8 questions.

H2: [Conclusion / What to do next]
    → 100-200 words. Clear CTA.
```

### How-To Tutorial

Target: 1,500-2,500 words

```
H1: How to [Action]: [Outcome or qualifier]

H2: What you'll need / Prerequisites
    → 100-150 words. Set expectations up front.

H2: [Step 1: Action verb + outcome]
    → One action per step. Include what to do if something goes wrong.

H2: [Step 2: Action verb + outcome]

H2: [Step 3: Action verb + outcome]
    → Continue for each step. 150-300 words per step.

H2: Tips and variations
    → 200-300 words. Advanced techniques, shortcuts, alternatives.

H2: Common mistakes to avoid
    → 200-300 words. Prevent the reader from failing.

H2: Frequently Asked Questions
    H3: [Question]
    H3: [Question]
    → 3-5 questions from PAA.

H2: Next steps
    → 100-150 words. CTA.
```

### Comparison

Target: 2,000-3,000 words

```
H1: [Option A] vs [Option B]: [Differentiator or use case]

H2: Quick verdict
    → 200-300 words. "Choose A if… Choose B if…"
    → Decision-maker who just wants the answer gets it here.

H2: [Feature/criteria] comparison table
    → 8-12 rows comparing specific features, pricing, capabilities.

H2: [Option A]: Full breakdown
    H3: Key features
    H3: Pros and cons
    H3: Best for
    → 600-800 words.

H2: [Option B]: Full breakdown
    H3: Key features
    H3: Pros and cons
    H3: Best for
    → 600-800 words.

H2: Head-to-head: [Scenario 1]
H2: Head-to-head: [Scenario 2]
    → 150-200 words each. Real-world use cases.

H2: Frequently Asked Questions
    → 3-5 questions from PAA.

H2: Final recommendation
    → 150-200 words. Clear opinion. CTA.
```

### Listicle

Target: 1,500-2,500 words

```
H1: [Number] Best [Category] [Qualifier — year, use case, audience]

H2: Quick summary
    → Comparison table with top picks and key differentiators.

H2: [Item 1: Name] — [One-line differentiator]
    → 150-250 words. What it is, why included, best for, limitations.

H2: [Item 2: Name] — [One-line differentiator]

H2: [Item 3: Name] — [One-line differentiator]
    → Continue for each item.

H2: How to choose [category]
    → 200-300 words. Decision framework.

H2: Frequently Asked Questions
    → 3-5 questions from PAA.

H2: Conclusion
    → 100-150 words. Top pick reiteration. CTA.
```

### News / Opinion Article

Target: 1,000-1,500 words (adjust from competitor analysis)

```
H1: [Topic]: [Angle or what we know]

H2: [Core story / what happened]
    → 200-400 words. Key facts, timeline, sources.

H2: [Context / background]
    → 200-300 words. What led to this, history, relevant background.

H2: [Impact / implications]
    H3: [Stakeholder or angle 1]
    H3: [Stakeholder or angle 2]
    → 300-500 words.

H2: [Reactions / what people are saying]
    → 150-200 words. Public, expert, or official responses.

H2: [What happens next]
    → 100-200 words. Forward-looking analysis.

H2: Frequently Asked Questions
    → 3-5 questions from PAA. Particularly valuable for news topics
    where people have many specific questions.
```

## Word Count Adjustments

The template targets above are defaults. Always prefer calculated
targets from competitor analysis:

1. Average the top 3-4 ranking competitors' word counts
2. Multiply by 1.2
3. Round to nearest 100
4. If the result is less than the template minimum, use the template minimum
5. If competitors vary wildly, use the top-performing competitor as baseline

Conciseness beats bloat. A 1,500-word article that covers everything
well will outperform a 3,000-word article padded with filler.
